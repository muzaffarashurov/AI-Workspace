# Execution Pipeline

Любая команда проходит одинаковый жизненный цикл.

---

1. User Command

Получение голосовой или текстовой команды.

↓

2. Intent Detection

Определение намерения пользователя.

↓

3. Planner

Создание плана выполнения.

↓

4. Task Manager

Создание новой задачи.

↓

5. Agent Selection

Выбор исполнителей.

↓

6. Execution

Выполнение задачи.

↓

7. Validation

Проверка результата.

↓

8. User Approval

Получение подтверждения пользователя (если требуется).

↓

9. Archive

Сохранение результатов.

---

Любая задача должна проходить через этот Pipeline.
