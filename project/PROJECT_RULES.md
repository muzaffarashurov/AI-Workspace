# PROJECT RULES

Эти правила обязательны для любого AI, который участвует в разработке проекта.

---

# Rule 1

Не менять архитектуру проекта без серьёзной причины.

---

# Rule 2

Не создавать дублирующий функционал.

---

# Rule 3

Сначала работающий код.

Потом новые функции.

Потом оптимизация.

---

# Rule 4

Любой новый код должен иметь одну ответственность.

---

# Rule 5

Любая интеграция должна быть заменяемой.

Запрещено жёстко привязывать систему к:

- OpenAI
- Anthropic
- Google
- Telegram
- GitHub

Все интеграции должны подключаться через отдельные модули.

---

# Rule 6

Orchestrator знает только роли.

Например:

Programmer

а не

Claude

или

GPT.

---

# Rule 7

Markdown является основной базой знаний.

Все инструкции должны храниться в Markdown.

---

# Rule 8

Любая задача должна быть оформлена как Task.

Task проходит через жизненный цикл.

---

# Rule 9

Каждый Commit должен делать систему лучше.

Не создавать пустые изменения.

---

# Rule 10

Если появляется выбор между:

красивым кодом

или

простым кодом

выбирать простой код.

---

# Rule 11

Не писать код "на будущее".

Создавать только то, что действительно понадобится в ближайших версиях.

---

# Rule 12

Перед добавлением новой функции необходимо ответить на вопрос:

Помогает ли эта функция приблизить систему к автономному выполнению работы пользователя?

Если нет — функцию следует отложить.
